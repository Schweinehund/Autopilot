   Functions to document (extract from source):
   - `Get-AutopilotDeviceStatus` — Gets comprehensive device state snapshot — No params — Returns hashtable with SerialNumber, HardwareHash, RegistrationState, ProfileAssignment, TPMStatus, NetworkConnectivity
   - `Get-AutopilotHardwareHash` — Retrieves device hardware hash via CIM — No params — Returns string (DeviceHardwareData) or null
   - `Get-AutopilotRegistrationState` — Checks Autopilot registration state from registry and WMI — No params — Returns hashtable: IsRegistered, CloudAssignedTenantId, CloudAssignedTenantDomain, DeploymentProfileName
   - `Get-AutopilotProfileAssignment` — Gets assigned Autopilot profile details from registry — No params — Returns registry property object or null
   - `Get-TPMStatus` — Checks TPM status and attestation capability — No params — Returns hashtable: TpmPresent, TpmReady, TpmEnabled, TpmActivated, TpmOwned
   - `Test-AutopilotConnectivity` — Tests connectivity to 5 required Autopilot endpoints — No params — Returns hashtable: URL to status string
   - `Get-AutopilotLogs` — Collects MDM diagnostics and Event Viewer logs — `-OutputPath` [string, default `$env:TEMP\AutopilotLogs`] — Returns string (output path)

6. `## Remediation Functions (AutopilotRemediation.psm1)`

   Document each of the 5 remediation functions using the same pattern, PLUS a safety warning callout.

   Functions:
   - `Reset-AutopilotRegistration` — Clears and re-registers device for Autopilot — `-Force` [switch] — ShouldProcess: Yes — Removes Diagnostics\Autopilot and AutopilotSettings registry keys
   - `Reset-TPMForAutopilot` — Clears TPM and prepares for re-attestation — No params — ShouldProcess: Yes — Calls Clear-Tpm -Force
   - `Repair-AutopilotConnectivity` — Fixes common network connectivity issues blocking Autopilot — No params — ShouldProcess: **No** — Resets WinHTTP proxy, flushes DNS, resets Winsock
   - `Restart-EnrollmentStatusPage` — Restarts enrollment process for stuck ESP — No params — ShouldProcess: Yes — Stops EnrollmentUX, removes FirstSync keys, triggers EnterpriseMgmt tasks
   - `Remove-AutopilotDevice` — Removes device from Autopilot completely — `-IncludeMDM` [switch] — ShouldProcess: Yes — Calls Reset-AutopilotRegistration; optionally removes all enrollment keys

   Safety warning format for remediation functions that support ShouldProcess (4 of 5):
   ```
   > **Warning:** This function modifies device state. Test in non-production first. Supports `-WhatIf` for dry run.
   ```

   CRITICAL: `Repair-AutopilotConnectivity` does NOT support ShouldProcess/-WhatIf. Its safety warning must instead read:
   ```
   > **Warning:** This function modifies network state (WinHTTP proxy, DNS cache, Winsock). It does NOT support `-WhatIf`. Test in non-production first.
   ```
