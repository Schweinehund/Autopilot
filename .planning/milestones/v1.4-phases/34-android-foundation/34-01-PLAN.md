    1. **### Work Profile** (under Ownership & Management Scope). Definition: "Android Enterprise's containerized partition on a user's device. The work profile creates a separate OS-level container that holds managed apps, data, and policies, isolated from the personal profile. Work profile apps appear on the launcher with a badged briefcase icon. Applies to both BYOD Work Profile (personally-owned device) and Work Profile on Corporate-Owned devices (WPCO, formerly COPE)." Cross-platform note: Windows "Work profile on personally-owned devices" applies only to Android; iOS has no work-profile equivalent — closest parallel is Account-Driven User Enrollment which uses managed APFS volume; macOS has no equivalent.

    2. **### Supervision** (under Ownership & Management Scope). **CALLOUT-ONLY ENTRY per D-11 and PATTERNS.md lines 81-86** — NO prose definition, only the callout:
       ```markdown
       ### Supervision

       > **Android note:** "Supervision" is an iOS/iPadOS management-state concept (see [Apple Glossary — Supervision](_glossary-macos.md#supervision)). Android does not use this term. The closest analog is [Fully Managed](#fully-managed), which is an ownership-mode designation, not a permanent device state. Do not conflate.
       ```

       Rationale: AEAUDIT-04 forbids "supervision" as an Android management term. Appears in alphabetical index so readers arriving from iOS mental model find the disambiguation.

    3. **### User Enrollment** (under Enrollment). Definition: Android user enrollment refers to AOSP user-associated enrollment where a Microsoft Entra user is tied to the device at enrollment. Cross-platform note: On iOS, "User Enrollment" refers to Apple's privacy-preserving BYOD method (Profile-based or Account-Driven) — a completely different concept. Link to `_glossary-macos.md#account-driven-user-enrollment`.

    4. **### Dedicated** (under Ownership & Management Scope). Definition: Android Enterprise Dedicated mode (a.k.a. COSU — Corporate-Owned Single-Use). Device operates in a locked kiosk state running one or more pre-selected apps via Managed Home Screen. Cross-platform note: Windows has Assigned Access / Shared Device mode; iOS has Shared iPad / Single App Mode — similar intent, different implementations.

    5. **### Corporate Identifiers** (under Provisioning Methods). Definition: Pre-loaded IMEI / serial number / MEID lists uploaded to Intune to tag devices as corporate-owned at enrollment. Cross-platform note: On iOS, ABM serial-number enrollment is the analog (enrolled via Apple Business Manager); on Windows, hardware hash upload for Autopilot. **Critical content:** State that Android 12+ removed IMEI/serial access from personally-owned work profile devices (BYOD), so corporate identifiers work only on Android 11 and earlier for BYOD — see `03-android-version-matrix.md#android-12-breakpoint`.

    6. **### COBO** (under Ownership & Management Scope). Definition: Corporate-Owned, Business-Only — Android Enterprise's fully managed mode for corporate-owned devices with no personal profile. Maximum admin control. Cross-platform note: No exact Windows/macOS/iOS term; closest is macOS ADE supervised corporate-owned or iOS ADE supervised.

    7. **### COPE** (under Ownership & Management Scope). Definition: Corporate-Owned, Personally-Enabled — older Android mode combining a fully managed device with a work profile for personal use. **CRITICAL: Use "Google recommends WPCO" language — do NOT write "COPE deprecated"** (per Pitfall 4, STATE.md research flag). Cross-platform note: No equivalent on Windows/macOS/iOS. Also add internal cross-ref to `#wpco`.

    8. **### BYOD** (under Enrollment). Definition: Bring-Your-Own-Device — Android Enterprise's personally-owned work profile enrollment. User installs Company Portal, which provisions a work profile container. Cross-platform note: Windows uses MAM-WE or device enrollment for BYOD; iOS uses Account-Driven User Enrollment or profile-based User Enrollment; macOS has limited BYOD support.

    9. **### DPC** (under Provisioning Methods). Definition: Device Policy Controller — the on-device app that applies Android Enterprise policies. In Intune-managed scenarios, the Microsoft Intune app is the DPC for fully managed / dedicated / corporate-owned work profile; Company Portal was the legacy DPC for BYOD. Cross-platform note: On iOS, the MDM profile itself acts as the policy controller (no on-device agent equivalent); on Windows, the MDM client is built into the OS; on macOS, similar built-in MDM client.

    10. **### Managed Google Play** (under Portals & Binding). Definition: Google's enterprise app distribution surface, tenant-bound to Intune. Used for approving, assigning, and distributing apps to Android Enterprise devices. Preferred binding since August 2024 uses Microsoft Entra account (not Gmail). Cross-platform note: On iOS, Volume Purchase Program (VPP) / Apps and Books is the analog; on Windows, Microsoft Store for Business (deprecated) or Winget is closest.

    11. **### afw#setup** (under Provisioning Methods). **USE EXPLICIT ANCHOR TAG per Assumption A2**:
        ```markdown
        <a id="afw-setup"></a>
        ### afw#setup
        ```
        Definition: Android for Work Setup identifier (historical but still functional) — a literal string entered in place of a Google account during device setup to trigger the Android Enterprise DPC download for fully managed and dedicated mode provisioning. Cross-platform note: No Windows/macOS/iOS equivalent; this is Android-specific provisioning signaling.

    12. **### WPCO** (under Ownership & Management Scope). Definition: Work Profile on Corporate-Owned devices — newer Android Enterprise mode combining a fully managed device with a user-separated work profile. Google's recommended successor pattern to COPE. Cross-platform note: No Windows/macOS/iOS equivalent; link back to `#cope` internally.

    13. **### Fully Managed** (under Ownership & Management Scope). Definition: Android Enterprise's highest-control enrollment mode where the entire device is managed — no personal profile, all apps and data corporate, full admin control over restrictions and configuration. The Android analog for COBO enrollment. Cross-platform note: The closest iOS analog is Supervised mode (a permanent management state), but mapping is partial — iOS supervision is a per-device state gating specific restriction settings; Android Fully Managed is an ownership-mode designation. See [_glossary-macos.md#supervision](_glossary-macos.md#supervision).

    **6 Android-Native Terms per D-12** (cross-platform note optional; include if it aids disambiguation):

    14. **### Zero-Touch Enrollment** (under Enrollment). Definition: Corporate-scale Android provisioning where OEM resellers upload IMEI / serial lists to the [Zero-Touch portal](https://enterprise.google.com/android/zero-touch/customers); devices enroll automatically on first boot. Reseller relationship is Step 0. Cross-platform note: Analog is iOS/iPadOS Automated Device Enrollment (ADE) via Apple Business Manager; Windows Autopilot is the Windows analog.

    15. **### Play Integrity** (under Compliance & Attestation). Definition: Google's device-attestation API (successor to SafetyNet, which Google turned off January 2025). Returns three verdict levels: Basic, Basic + Device, and Strong integrity (requires hardware-backed security). Intune compliance UI uses "Play Integrity Verdict" terminology. Cross-platform note: On iOS, DeviceCheck / Device Attestation serves a similar purpose; on Windows, TPM attestation.

    16. **### AMAPI** (under Compliance & Attestation). Definition: Android Management API — Google's newer management API. Microsoft migrated Android personally-owned work profile (BYOD) management to AMAPI in April 2025. Custom OMA-URI profiles for BYOD are no longer supported. Wi-Fi configuration requires certificate-based authentication (username/password Wi-Fi broken). The Microsoft Intune app is the primary BYOD management surface post-migration.

    17. **### Managed Home Screen** (under Compliance & Attestation). Definition: Microsoft's launcher app for Android Enterprise Dedicated (COSU) devices. Enforces kiosk mode, app allow-list, and exit-kiosk PIN. Exit-PIN synchronization requirement — the PIN configured in the device-restrictions profile must match the PIN in the Managed Home Screen app config, or users hit a visible error (see `03-android-version-matrix.md` and Phase 38).

    18. **### Entra Shared Device Mode** (under Portals & Binding). Definition: Microsoft Entra authentication mode for shared Android devices. Users sign in / sign out via supported line-of-business apps; Entra state is scoped to the active user session. Used on Dedicated devices in shift-worker scenarios.

    19. **### EMM** (under Portals & Binding). Definition: Enterprise Mobility Management — the umbrella term for MDM-plus-app-management platforms. Microsoft Intune is an EMM; Google's Android Enterprise program certifies EMMs via the Managed Google Play bind.
