**## How to Use This Guide** -- Audience routing section (per established pattern from macOS lifecycle). Route L1 to the comparison table and per-path sections, L2 to the Supervision section, and admins to the selection guidance in each per-path section.

**## The Four iOS/iPadOS Enrollment Paths** -- Introductory narrative of 2-3 paragraphs (per D-01) establishing the iOS enrollment landscape. Must state:
- iOS/iPadOS supports four distinct enrollment approaches with fundamentally different management scopes
- The choice of enrollment path determines supervision state, which is permanent once set
- Three paths involve MDM enrollment (device management profile installed); one path (MAM-WE) operates at the app layer only with no device enrollment

**## Enrollment Path Comparison** -- The SC #1 comparison table (per D-02, D-14) with exactly these columns and values:

| Enrollment Path | Ownership Model | Supervision State | Management Scope | Appropriate Use Case |
|-----------------|-----------------|-------------------|------------------|---------------------|
| Automated Device Enrollment (ADE) | Corporate-owned | Supervised | Full device (all MDM capabilities including supervised-only restrictions, remote wipe, OS update enforcement, silent app install) | Corporate fleet; zero-touch deployment via ABM; new or wiped devices only |
| Device Enrollment | Personal or corporate | Unsupervised | Device (MDM-managed policies, certificates, Wi-Fi, VPN, compliance -- no supervised-only capabilities) | BYOD or legacy corporate devices not eligible for ADE; no ABM required |
| User Enrollment (account-driven) | Personal (BYOD) | Unsupervised | App-layer + limited device (separate managed APFS volume; per-app VPN, Wi-Fi; no UDID, serial, or IMEI collection) | Privacy-preserving BYOD; iOS 15+ required |
| MAM Without Enrollment (MAM-WE) | Any | N/A (no enrollment) | App-layer only (policy controls within SDK-integrated apps; no MDM profile installed) | Unmanaged personal devices needing M365 app access; contractor/partner devices |

**## Supervision** -- Dedicated section (per D-12, D-13) with three subsections or clearly delineated paragraphs:
1. **What supervision is:** A management state set at enrollment time. When a device is supervised, the MDM server has expanded control. Unsupervised devices have standard MDM management only.
2. **When supervision is set:** At enrollment time, exclusively through ADE. For iOS/iPadOS 13.0 and later, devices enrolled via ADE are automatically placed in supervised mode. Supervision cannot be added to a device enrolled without it.
3. **Changing supervision requires full device erase:** State explicitly: "Changing a device from unsupervised to supervised requires a full device erase and re-enrollment via ADE." Use the phrase "full device erase" (NOT "wipe" or "selective wipe"). Note that a full device erase removes ALL data including personal data -- this is not a selective wipe. Include verification location: Settings > General > About shows "This iPhone is supervised and managed by [organization name]" on supervised devices.
4. **Forward reference (D-15):** End with: "Subsequent admin setup guides mark supervised-only settings with the supervised-only callout pattern."

**### Automated Device Enrollment (ADE)** -- Per-path section (per D-04, anchor: `#automated-device-enrollment-ade`). 3-5 sentences covering: zero-touch via ABM, supervised mode, corporate ownership model, link to ADE lifecycle doc (`01-ade-lifecycle.md`). Include inline supervision callout linking back to `#supervision` anchor.

**### Device Enrollment** -- Per-path section (anchor: `#device-enrollment`). 3-5 sentences covering: Company Portal or web-based enrollment, unsupervised, personal or corporate devices, no ABM required. Include inline supervision callout noting unsupervised state.

**### User Enrollment** -- Per-path section (anchor: `#user-enrollment`). 3-5 sentences covering: account-driven enrollment (NOT profile-based -- deprecated iOS 18), privacy-preserving BYOD, separate managed APFS volume, iOS 15+ required.
