**Step 1 — Run the GFM capital-anchor precheck:**

```bash
mkdir -p /tmp/phase-48
grep -rnE '#[A-Z]' docs/ > /tmp/phase-48/capital-anchors.txt 2>/dev/null
wc -l /tmp/phase-48/capital-anchors.txt
```

The output captures every line containing `#X` where X is an uppercase A-Z. Two interpretation considerations:

A. **Markdown link references**: `[text](#Anchor)` and `[text](relative-path.md#Anchor)` — both indicate anchor link references that may have GFM-normalization issues.

B. **Heading text false positives**: `## Section #1` (where the `#1` is part of heading text) — these are NOT broken links, but the grep matches them. Filter manually.

Filter the precheck output to extract only link references (`](#[A-Z]` and `](.*#[A-Z]` patterns):

```bash
grep -E '\]\(#[A-Z]|\]\([^)]*#[A-Z]' /tmp/phase-48/capital-anchors.txt > /tmp/phase-48/capital-anchor-links.txt
wc -l /tmp/phase-48/capital-anchor-links.txt
```

If the filtered count is large (>50), spot-check 5 entries to confirm they are link references not heading-noise.

**Step 2 — For each line in the filtered output, parse: file:line:content. Extract link target via regex match.**

```
docs/admin-setup-android/04-cobo-knox-me.md:42:See [Knox Setup](#KnoxMobileEnrollment) for details.
```

Parses to:
- file: `docs/admin-setup-android/04-cobo-knox-me.md`
- line: `42`
- link target: `#KnoxMobileEnrollment`
