```javascript
{
  id: 12,
  name: 'C12: 4-platform comparison structural validation',
  informational: true,
  // AUDIT-04: INFORMATIONAL-FIRST. File-existence pre-gate. Promotes to blocking once
  // docs/reference/4-platform-capability-comparison.md exists (Phase 58+).
  // Link-not-copy: every non-empty data cell must contain a markdown hyperlink.
  run() {
    const targetFile = 'docs/reference/4-platform-capability-comparison.md';
    const content = readFile(targetFile);
    if (!content) {
      // File doesn't exist yet (Phase 48-57) -- informational PASS (pre-gate)
      return { pass: true, detail: '(informational)' };
    }
    // File exists (Phase 58+) -- compute structural diagnostics but always return informational PASS.
    // Phase 60 graduation flips these to { pass: false, detail: ... }.
    const platforms = ['Windows', 'macOS', 'iOS', 'Android', 'Linux'];
    const missingPlatforms = platforms.filter(p => !content.includes(p));
    const tableLines = content.split('\n').filter(l => /^\|/.test(l) && !/^\|[-: ]+\|/.test(l));
    let emptyCellCount = 0;
    for (const line of tableLines) {
      const cells = line.split('|').slice(1, -1);
      for (const cell of cells) {
        const trimmed = cell.trim();
        if (trimmed && trimmed !== '—' && trimmed !== 'N/A' && !/\[.+\]\(.+\)/.test(trimmed)) {
          emptyCellCount++;
        }
      }
    }
    return { pass: true, detail: '(informational)' };
  }
},
```

Exact check object to insert (verbatim from 48-RESEARCH.md):

```javascript
{
  id: 13,
  name: 'C13: Broken-link automation (markdown-link-check)',
  informational: true,
  // AUDIT-05: INFORMATIONAL-FIRST. Promotes to blocking after Phase 60 second-pass triage.
  // Scope: internal anchor links + relative paths in docs/**/*.md.
  // External MS Learn URL validation explicitly OUT OF SCOPE (REQUIREMENTS.md).
  async run() {
    const docFiles = [];
    for (const abs of walkMd('docs')) { docFiles.push(relNormalize(abs)); }
    if (docFiles.length === 0) return { pass: true, detail: '(informational)' };
    try {
      const { execFileSync } = await import('node:child_process');
      execFileSync('npx', [
        '--yes', '--no-install', 'markdown-link-check',
        '--config', '.mlc-config.json',
        '--quiet',
        ...docFiles.slice(0, 10)
      ], { stdio: 'pipe', timeout: 60000, cwd: process.cwd() });
      return { pass: true, detail: '(informational)' };
    } catch (err) {
      // Always informational-first PASS regardless of tool output or ENOENT
      return { pass: true, detail: '(informational)' };
    }
  }
},
```

Critical caveat — async run() compatibility: v1.4.1 harness runner at line ~370-385 uses synchronous iteration (`const result = check.run()`). If C13's run is async, the runner receives a Promise instead of `{ pass, detail }`. There are two ways to address this:

Option A (preferred — minimal runner change): Update the runner loop to `await` each check.run(). Find the runner loop block in v1.4.1 lines 367-391; change `const result = check.run()` to `const result = await check.run()`. Wrap the surrounding iteration in an async IIFE: change the bottom-of-file pattern to `(async () => { ... existing runner code ... })().catch(err => { console.error(err); process.exit(2); });`.

Option B (synchronous C13): Use `require('node:child_process').execFileSync` synchronously without dynamic import. This requires changing C13 to a non-async function and using a top-of-file `import { execFileSync } from 'node:child_process';` (Path A modification — allowed per Phase 43 D-06 additive precedent).

Decision: Use Option A. Reason: Option A keeps imports clean and follows the dynamic-import pattern in 48-RESEARCH.md. The async-runner change is small (2 lines) and structurally invisible to existing checks (C1-C7, C9, C10, C11, C12 are all sync; awaiting a sync function returns the value directly).
