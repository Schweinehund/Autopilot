**Step 2 — Write the file with exact content (verbatim from 48-RESEARCH.md):**

```json
{
  "ignorePatterns": [
    { "pattern": "^https://learn\\.microsoft\\.com" },
    { "pattern": "^https://docs\\.microsoft\\.com" },
    { "pattern": "^https://techcommunity\\.microsoft\\.com" },
    { "pattern": "^https://portal\\.azure\\.com" },
    { "pattern": "^https://endpoint\\.microsoft\\.com" },
    { "pattern": "^https://intune\\.microsoft\\.com" }
  ],
  "retryOn429": true,
  "retryCount": 2,
  "timeout": "10s",
  "aliveStatusCodes": [200, 201, 206, 301, 302, 303, 307, 308]
}
```

**Field-by-field rationale (for SUMMARY documentation):**
- `ignorePatterns`: 6 Microsoft-domain regex patterns covering all MS-controlled URL spaces referenced across the docs corpus. Excludes from sweep per REQUIREMENTS Out-of-Scope.
- `retryOn429: true` + `retryCount: 2`: GitHub/npm/raw CDN links may return 429 transiently. Retry up to 2 times before flagging as broken.
- `timeout: "10s"`: Per-link request budget. Internal-link checks complete in milliseconds; this only matters for any external links not caught by ignorePatterns (rare).
- `aliveStatusCodes`: Includes 301/302/303/307/308 redirect codes — these are "alive" links from the perspective of broken-link detection. Combined with ignorePatterns, the redirect-chain noise from MS Learn is fully neutralized.
