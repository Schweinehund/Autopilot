## Tech Debt Detail

### Phase 5 — L1 Runbooks (warning)

**Generic escalation links in 3 of 5 L1 runbooks.**

`L1RB-01`, `L1RB-03`, and `L1RB-05` escalate to `l2-runbooks/00-index.md` rather than a specific receiving L2 runbook. This is functionally correct (L2 can navigate from the index) but adds a navigation step during an active incident. In the L1RB-05 OOBE case, the escalation does not mention that log-collection is a mandatory prerequisite before investigation runbooks.

**Suggested fix:** Update each escalation block with:
- L1RB-01: "Start with `l2-runbooks/01-log-collection.md`, then `l2-runbooks/04-hybrid-join.md` if hybrid-joined"
- L1RB-03: "Route based on symptom — `l2-runbooks/02-esp-deep-dive.md` if ESP hangs, `l2-runbooks/05-policy-conflicts.md` if policies stuck in pending"
- L1RB-05: "Run `l2-runbooks/01-log-collection.md` first, then investigate via `l2-runbooks/00-index.md`"

### Phase 7 — Navigation (warning)

**Hardware hash is not a standalone entry in `common-issues.md`.** It is covered under "Device Registration Issues" but keyword search for "hardware hash" will not find a direct hit. This is a discoverability issue, not a broken link.

**Quick-ref cards are reachable only from `index.md`.** `quick-ref-l1.md` and `quick-ref-l2.md` have one inbound link each (from the master index). Staff already inside a runbook cannot reach the quick-ref without navigating back to the index. During an active incident this adds friction.

**Suggested fix:** Add a persistent "Quick reference" footer link to each runbook template and regenerate the runbooks, or add a header nav row linking to the quick-ref card for the runbook's audience.

### Phase 6 — L2 Runbooks (info)

**`l2-runbooks/01-log-collection.md` has no error-code cross-reference.** Architecturally defensible: log-collection is procedural, not error-code-driven. However, there is no reverse path from the error-code tables back to log-collection, so L2 staff following an error code to a runbook may miss the log-collection prerequisite.

### Phase 1 — Foundation (info)

**`_templates/l1-template.md` and `_templates/l2-template.md` are orphaned.** Expected — these are authoring scaffolds, not user-facing. Noted for completeness; no action required.
