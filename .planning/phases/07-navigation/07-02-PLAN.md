**## Top 5 Checks** — numbered checklist per D-06. These map to TRD1-TRD6 in the initial triage tree:

1. **Device in portal?** — Intune > Devices > Windows > Enrollment > Windows Autopilot devices — search by serial number
2. **Profile assigned?** — Same portal page, Profile column shows "Assigned" (not blank or "Not assigned")
3. **Endpoints reachable?** — From device browser, navigate to `https://login.microsoftonline.com` — page loads without error
4. **ESP past expected time?** — Device phase: >30 min stuck. User phase: >60 min stuck.
5. **Error code on screen?** — Look up in [Error Code Index](error-codes/00-index.md)

**## Escalation Triggers** — compact bullet list per D-06. Extracted from all 5 L1 runbook escalation criteria sections (read each runbook's escalation section and condense to bullet fragments):

- Serial confirmed correct, device not in portal → **Escalate L2** (collect: serial number, hardware hash screenshot, tenant ID)
- ESP stuck at same point after one restart AND over time threshold → **Escalate L2** (collect: error code if visible, screenshot, time elapsed)
- Error code not in lookup table or L1 action did not resolve → **Escalate L2** (collect: exact error code, screenshot)
- Device in correct group, profile still not assigned after 30 min → **Escalate L2** (collect: device serial, group name, profile name expected)
- Wrong profile assigned → **Escalate L2** (collect: current profile name, expected profile name)
- Cannot reach login.microsoftonline.com or Autopilot endpoints → **Escalate Infrastructure/Network** (NOT L2)
- OOBE fails a second time with same symptoms → **Escalate L2** (collect: exact error, screenshot, number of attempts)
- Blue screen or Windows recovery mode → **Escalate L2** (collect: stop code if visible, device model)

**## Log Collection** — per D-07, exact copy-pasteable command:

```
mdmdiagnosticstool.exe -area Autopilot -cab -out C:\Temp\AutopilotDiag
```

Registry snapshot command:
```
reg export "HKLM\SOFTWARE\Microsoft\Windows\Autopilot\EnrollmentStatusTracking" "%COMPUTERNAME%_%DATE:~-4%-%DATE:~4,2%-%DATE:~7,2%_esp-registry.reg" /y
```

**## PowerShell Diagnostic Commands** — per D-07, top 6 with exact syntax. Ranked by frequency of appearance in L2 runbooks:

```powershell
Get-AutopilotDeviceStatus                    # Comprehensive device snapshot
Get-AutopilotRegistrationState               # Tenant ID and profile confirmation
Get-TPMStatus                                # TPM readiness check
Test-AutopilotConnectivity                   # Endpoint reachability test
Get-AutopilotLogs -OutputPath "C:\Temp\AutopilotDiag\$env:COMPUTERNAME"  # One-command log collection
Get-AutopilotHardwareHash                    # Hardware hash retrieval
```

**## Event Viewer Log Paths** — per D-07, 4 literal strings:

**## Registry Paths** — per D-07, top 4 with when-to-use:

| Path | When to Use |
|------|------------|
| `HKLM:\SOFTWARE\Microsoft\Provisioning\Diagnostics\Autopilot` | Registration state: TenantId, TenantDomain, ProfileName |
| `HKLM:\SOFTWARE\Microsoft\Provisioning\AutopilotSettings` | Profile assignment details |
| `HKLM:\SOFTWARE\Microsoft\Enrollments\{GUID}\FirstSync` | ESP device phase tracking (IsServerProvisioningDone: 0=device, 1=done) |
| `HKLM:\SOFTWARE\Microsoft\Windows\Autopilot\EnrollmentStatusTracking` | ESP root — parent of all ESP diagnostic data |

**## Key Event IDs** — per D-07, grouped by scenario:

| Scenario | Event IDs |

| TPM attestation | 171 (TPMIdentityFailed), 172 (AutopilotProfileUnavailable) |
| ESP/enrollment | 0x81036501 (MDMInfoNotFound), 0x81036502 (AppInstallFailure) |
