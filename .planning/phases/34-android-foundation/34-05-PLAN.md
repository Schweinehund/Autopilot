<objective>
Create `docs/_templates/admin-template-android.md` — the tri-portal admin guide template that Phase 35-39 admin setup guides (Managed Google Play binding, Zero-Touch portal, Fully Managed COBO, BYOD Work Profile, Dedicated devices, AOSP stub) will copy as their starting point (AEBASE-05 / ROADMAP Success Criterion 3).

Purpose: All Android admin setup guides must use the SAME shape so readers navigate consistently. iOS and macOS templates established a two-portal pattern (ABM + Intune admin center). Android requires a three-portal pattern because Managed Google Play binding and Zero-Touch portal configuration are BOTH prerequisite gates that admins configure outside Intune. Without this template, Phase 35-39 authors would invent shapes — leading to drift.

Output: One new markdown file at `docs/_templates/admin-template-android.md` containing: author-instruction HTML comment header, frontmatter, platform gate blockquote, callout-pattern comment, Prerequisites section, Steps section with 3 H4 portal sub-sections (one of them wrapped in subtractive-deletion HTML comment per D-17), Verification section, Configuration-Caused Failures table, Renewal/Maintenance section (mandatory-by-default per D-20), See Also.

This plan MUST honor 7 locked decisions: D-16 (3 H4 sub-sections with exact names), D-17 (ZT subtractive-deletion HTML comment VERBATIM wording locked), D-18 (MGP is mandatory — NOT wrapped in subtractive comment), D-19 (What breaks if misconfigured callouts required per configurable setting), D-20 (Renewal/Maintenance mandatory by default, unlike iOS/macOS), D-21 (frontmatter with 60-day review cycle + `platform: Android` literal), D-22 (template is not numbered in admin-setup-android/ sequence — lives at `docs/_templates/admin-template-android.md`).
</objective>
