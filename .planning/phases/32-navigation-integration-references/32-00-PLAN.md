Create the directory `.planning/phases/32-navigation-integration-references/validation/` and 5 files:

**1. `link-check.sh`** (POSIX bash, shebang `#!/usr/bin/env bash`, `set -euo pipefail`):
- Accepts 1 optional arg (single file path or directory; default `docs`).
- For each `.md` file under the scan target (excluding `_templates/**`):
  - Use `grep -nE '\[[^]]+\]\(([^)]+\.md)(#[^)]+)?\)'` to extract links with line numbers.
  - For each link, resolve target path relative to containing file's directory.
  - If target file does not exist → emit `BROKEN: <file>:<line> -> <target> (file not found)`, set EXIT_CODE=1.
  - If link has a fragment anchor (`#...`): slugify H2/H3 headings in target file (lowercase, spaces→hyphens, strip non-alphanumeric except `-`) and verify anchor exists. If not → emit `BROKEN: <file>:<line> -> <target>#<anchor> (anchor not found)`, set EXIT_CODE=1.
- Exit with EXIT_CODE (0 if clean, 1 if any broken links).
- Slugification MUST handle:
  - Lowercase conversion
  - Spaces → `-`
  - Strip `:`, `/`, `(`, `)`, `,`, `.`
  - Preserve `-` and alphanumerics
  - Collapse multiple `-` to single `-`

**2. `anchor-collision.sh`** (POSIX bash, shebang + `set -euo pipefail`):
- Accepts 1 optional arg (single file or directory; default `docs`).
- For each `.md` file (excluding `_templates/**`):
  - Extract H2 and H3 heading text: `grep -E '^#{2,3} ' | sed 's/^#\+ //'`.
  - Find duplicates: `sort | uniq -d`.
  - If any duplicates → emit `COLLISION in <file>:` followed by the duplicate heading texts (indented), set EXIT_CODE=1.
- Exit with EXIT_CODE.
- Expected Phase 32 state: exit 0 after iOS sections use `iOS: ` prefix per D-22 disambiguation pattern (RESEARCH.md Pitfall 1).

**3. `reachability-audit.sh`** (POSIX bash, BFS from `docs/index.md`):
- ROOT defaults to `docs/index.md`; MAX_DEPTH defaults to 2.
- Maintain associative array `VISITED[file]=depth:path` and FIFO queue.
- BFS: dequeue entry → extract `.md` links from file → for each unvisited target, set VISITED[target]="(depth+1):<path>" and enqueue if depth+1 < MAX_DEPTH (always record, traverse only if below MAX_DEPTH).
- Output: one line per visited file as `<file> | <depth> | <path-taken>`, sorted by depth ascending.
- Relative link resolution: strip fragment, resolve relative to containing file's directory; normalize via `realpath --relative-to=. ...` or equivalent portable construct. If `realpath` unavailable, use shell path-normalization function.
- Exclude `_templates/**` and any `.planning/` paths from traversal.
- Exit 0 always (script reports; the orchestrator compares against fixture).

**4. `run-all.sh`** (POSIX bash orchestrator):
- `set -euo pipefail`.
- Determine HARNESS_DIR via `$(cd "$(dirname "$0")" && pwd)`.
- Run sequentially:
  1. `echo "=== link-check ==="` then `"$HARNESS_DIR/link-check.sh" "docs"` (exit on failure via `set -e`).
  2. `echo "=== anchor-collision ==="` then `"$HARNESS_DIR/anchor-collision.sh" "docs"`.
  3. `echo "=== reachability-audit ==="` then `"$HARNESS_DIR/reachability-audit.sh" | diff - "$HARNESS_DIR/expected-reachability.txt"`.
- `echo "=== all checks passed ==="` on success.
