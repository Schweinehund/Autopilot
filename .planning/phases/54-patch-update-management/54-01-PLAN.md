```markdown
## Ring Terminology {#ring-terminology}

The word "ring" is overloaded in Microsoft Windows update tooling. **WUfB deployment ring** and
**Autopatch ring** are mutually exclusive concepts; they cannot coexist on the same device. This
section disambiguates them and the separate driver/firmware update policy:

- **WUfB deployment ring** — A Windows Update for Business (WUfB) Update ring policy in Intune
  configures deferral periods, deadlines, and restart behavior for quality and feature updates.
  Tenants assign devices to WUfB deployment rings via Intune > Devices > Update rings, and devices
  receive Microsoft updates directly per the ring's configured deferral cadence.
- **Autopatch ring** — A Windows Autopatch ring is a service-managed device cohort (Test, First,
  Fast, Broad rings) that Windows Autopatch automatically rotates and gates. Autopatch is mutually
  exclusive with WUfB deployment rings; enabling Autopatch detaches devices from any pre-existing
  WUfB deployment ring assignment.
- **Driver/firmware update policy** — Separately configured under Intune > Devices > Windows >
  Driver and firmware updates; this is NOT a "ring" in either WUfB or Autopatch sense. It is a
  discrete update policy surface and follows its own approval cadence.

For Windows-specific WUfB deployment ring and Autopatch ring configuration, including PITFALL-9
mutual-exclusion practical guidance, see [Windows WUfB Rings](01-windows-wufb-rings.md).
```

**Section 6 — Deferral vs Enforcement terminology H2 (V-54-10; cross-platform terminology):**

```markdown
## Deferral vs Enforcement

Patch management uses two different control primitives across platforms:

- **Deferral** — A tenant-side delay applied before an update is installed. The user/device is told
  "this update exists; you may install later." Examples: Windows WUfB deferral periods (0-30 days
  quality; 0-365 days feature); macOS legacy MDM `forceDelayedSoftwareUpdates`; iOS legacy
  device-restrictions "Defer software updates" (supervised-only). Deferral is a soft control:
  the user can usually install earlier; the deferral expires.
- **Enforcement** — A tenant-side hard requirement that the OS install an update by a specific
  deadline. Examples: Windows WUfB deadline + grace period (after deadline, force-install +
  reboot); macOS DDM "Software Update Enforce Latest" (declarative — OS enforces target version);
  iOS DDM update keys (TargetOSVersion, TargetBuildVersion, TargetLocalDateTime, OfferPrograms);
  Android Play Integrity MEETS_STRONG_INTEGRITY compliance gate (devices not patched within
  12 months fail compliance).
- **Attestation** — A check that the device is currently patched to a recent baseline. Used for
  Conditional Access. Example: Android Play Integrity verdicts; iOS device-level encryption check.

Modern Apple platforms (Apple OS 26 forward) are deprecating deferral primitives in favor of DDM
enforcement. See per-platform guides for migration timelines.
```
