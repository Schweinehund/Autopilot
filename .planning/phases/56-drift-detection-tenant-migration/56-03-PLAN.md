```markdown
# macOS Drift Detection: Profile Revocation + DDM Compliance Signals

This guide is a macOS-specific drift detection summary. **macOS does not have a native Intune
Remediations equivalent** — Intune Remediations (Proactive Remediations) is currently
Windows-only. macOS-side compliance drift surfaces through three primary signal classes covered
below: profile revocation (when MDM trust breaks or a configuration profile is removed), DDM
declaration verdicts (when DDM declarations are deployed and the device evaluates them), and
Intune Settings Catalog drift signals (per-policy compliance state).

For the cross-platform compliance drift overview and signal comparison table, see
[Drift Detection Overview](00-overview.md). For substantive macOS investigation procedures
(log collection, agent diagnostics, profile re-push), follow the v1.2 macOS L1 and L2 runbooks
linked in the Related Resources footer — those runbooks own the operational depth surface.
This guide provides only the cross-platform-routing summary.
```

**Section 4 — `## macOS Compliance Drift Signals` H2 (operational summary; ≤30 lines):**

```markdown
## macOS Compliance Drift Signals

Three macOS-side compliance drift signal classes that admins should be aware of:

- **Profile revocation** — When the MDM-pushed configuration profile is removed by the user
  (rare in supervised contexts; common in BYOD where the user has removable profile permissions)
  or when MDM trust expires (push token expired, certificate rotation lapsed). Surface: Intune
  compliance reports show the device as non-compliant for the policies that depended on the
  revoked profile. Remediation: re-push the affected configuration profiles, or retire and
  re-enroll the device for severe trust failures.
- **DDM (Declarative Device Management) declaration verdicts** — DDM declarations carry
  per-declaration status that the device reports back to Intune (`Active`, `Failed`, `Idle`).
  When a declaration transitions from `Active` to `Failed`, that is a drift signal indicating
  the device cannot satisfy the declaration's predicates. Surface: Intune Settings Catalog ->
  per-device declaration state.
- **Intune Settings Catalog drift signals** — Per-policy compliance state across the
  Settings Catalog macOS scope. Surface: Intune compliance reports + `DeviceNonCompliance`
  Graph export-job report.

**No native Intune Remediations equivalent.** Despite the cross-platform "Bash/PowerShell"
framing in some narratives, Intune Remediations runs only on Windows-managed devices. macOS
shell-script monitoring (Intune Settings Catalog macOS shell scripts) is a different surface
that does not include the detect+remediate-pair semantics; see Microsoft Learn macOS shell
scripts for that surface.
```
