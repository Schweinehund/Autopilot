# iOS + Android Drift Detection: Jailbreak, OS Downgrade, and Play Integrity Verdicts

This guide covers compliance drift signals for **iOS/iPadOS** (jailbreak detection + OS

signals through the Intune compliance policy framework plus platform-specific attestation

**Section 4 — `## iOS Drift Signals` H2 (operational summary; ≤20 lines):**

```markdown
## iOS Drift Signals

Two iOS-side compliance drift signal classes that admins should be aware of:

- **Jailbreak detection** — Intune iOS compliance policy includes a "Jailbroken devices"
  setting that, when set to **Block**, marks any device the platform reports as jailbroken
  as non-compliant. Surface: Intune compliance reports + per-device compliance state.
  Remediation: jailbroken iOS devices cannot be remediated to a healthy state via MDM —
  the user must restore the device via iTunes / Finder and re-enroll, OR the device must
  be removed from CA scope.
- **OS downgrade signal** — Intune iOS minimum-OS-version compliance policy flags devices
  that drop below the minimum version. Surface: Intune compliance reports. Note: iOS does not
  permit user-driven OS downgrades via standard channels; downgrade typically requires a full
  device restore from a backup made on an older iOS version, which then requires re-enrollment.

For substantive iOS investigation depth (jailbreak triage, profile re-push procedures,
configuration profile recovery), follow the v1.3 iOS L1 and L2 runbooks linked in the
Related Resources footer.
```

**Section 5 — `## Android Drift Signals` H2 (operational summary; ≤20 lines; PITFALL-7 link-not-copy):**

```markdown
## Android Drift Signals

The primary Android compliance drift signal is **Play Integrity verdict change**:

- A device that previously satisfied `MEETS_STRONG_INTEGRITY` may transition to a lower
  verdict (`MEETS_DEVICE_INTEGRITY` or `MEETS_BASIC_INTEGRITY`) due to security patch
  staleness, a system modification, or root detection. This drift is reported through Intune
  compliance policy with Play Integrity attestation enabled.
- Surface: Intune compliance reports + per-device Play Integrity attestation state.
- Remediation: apply pending security patches; for persistent verdict drops, the device may
  need to be re-imaged or removed from CA scope.

> **Substance lives in the Phase 54 SSoT.** For the full Play Integrity verdict-change drift
> story including the MEETS_STRONG_INTEGRITY enforcement timeline, Android 13+ verdict-change
> behavior, and security-patch-staleness mapping, see
> [Android Patch Delivery](../patch-management/04-android-patch-delivery.md). This guide
> cross-links rather than duplicating that substance per the cross-platform link-not-copy
> contract.
```
