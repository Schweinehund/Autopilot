They correspond to Stage 3 of the [Autopilot lifecycle](../lifecycle/03-oobe.md)."
5. Table legend block (same as 01-mdm-enrollment.md)
6. 7-column table header: `| Code | # | Name | Phase | Mode | Cause | L1 Action | L2 Fix |`
7. Error rows for 8 codes from RESEARCH.md ERRC-03 section:
   - 0x80070490, 0x800705b4, 0x80190190, 0x801c03ea (TWO rows: cause 1 = TPM 1.2 not upgraded, cause 2 = dual profile assignment), 0x801C03F3, 0x81039001, 0x81039023, 0x81039024
   - Sorted by hex code ascending
   - **Hardware-specific notes in Cause column** per CONTEXT.md locked decision:
     - 0x80070490: "AMD fTPM firmware — check OEM for updated version"
     - 0x800705b4: Include VM/TPM 1.2 incompatibility note for SD mode (per RESEARCH.md Pitfall 6)
     - 0x81039024: "TPM firmware has known security vulnerability"
   - L1 Action: TPM errors generally require escalation with BIOS version + TPM manufacturer in collection list. 0x81039001 is the exception: "Retry provisioning (max 2 attempts, 15 min between); then **Escalate**" per RESEARCH.md Pitfall 5.
   - L2 Fix: Forward-link to Phase 6 TPM runbook: `see [L2 TPM runbook](../l2-runbooks/tpm-attestation.md) (available after Phase 6)` where appropriate. Link to powershell-ref.md and registry-paths.md for actionable L2 fixes.
   - 0x801c03ea cause 2 (dual profile): L1 can check group membership in portal before escalating (per RESEARCH.md Open Question 4).
8. **Hardware-Specific Known Issues section** (H2 below the table): Brief subsection documenting the 5 hardware-specific TPM issues from RESEARCH.md:
   - Infineon SLB9672 (firmware 15.22 EK cert issue)
   - ST Micro and Nuvoton RSA-3072
   - AMD fTPM
   - Intel Tiger Lake (KB5007253 required)
   - General clock skew (w32tm /resync)
   This section provides the detail that the Cause column abbreviates. Keep each to 2-3 sentences. These are NOT L2 investigation procedures (those go in Phase 6).
9. Prev/next navigation: `Prev: [MDM Enrollment Errors](01-mdm-enrollment.md) | Next: [ESP and Enrollment Errors](03-esp-enrollment.md)`
10. APv2 Notes blockquote: Conservative — note that TPM attestation is not used in APv2 Device Preparation (which does not require PP/SD modes), so most codes in this file do not apply.
11. Version History table.

**First-mention glossary linking:** Link first occurrence of TPM, OOBE, pre-provisioning, self-deploying, ESP to glossary.

**CRITICAL:** The Hardware-Specific Known Issues section is what makes ERRC-03 satisfy the "hardware-specific notes" requirement. The Cause column in the table references these briefly; the section below expands. This is not an L2 runbook — keep it factual and brief.
