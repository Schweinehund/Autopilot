# v1.4.1 Pitfalls Research — Adding Knox ME + Full AOSP per-OEM + COPE Full Admin to Existing v1.4 Suite

**Domain:** Android Enterprise documentation completion — Knox Mobile Enrollment, AOSP per-OEM expansion (RealWear / Zebra / Pico / HTC VIVE Focus / Meta Quest), COPE full admin path (or deprecation-rationale doc), plus audit-harness and integration retrofits against the v1.4 suite that shipped 2026-04-24 with `tech_debt` accepted.
**Researched:** 2026-04-24
**Confidence:** HIGH (Samsung Knox docs, Google AE Help, MS Learn) / MEDIUM (community / RealWear support / Bayton) where noted

> **Scope discipline:** This file is the v1.4.1 pitfall surface. It does NOT re-catalog pitfalls already addressed in v1.4 (SafetyNet deprecation, COPE "deprecated" language ban, KME↔ZT top-level callout, AOSP PITFALL-7 framing, mode-first L1 triage deviation, 60-day freshness rule). Those are baseline. v1.4.1 pitfalls are pitfalls of **adding** to that baseline.

**Decision:** At plan-time for the COPE phase (DEFER-06), run a dedicated research gate BEFORE authoring begins. The research gate determines whether v1.4.1 ships:

1. **COPE full admin guide** — parallel structure to `03-fully-managed-cobo.md`: prerequisites, enrollment profile, token management, provisioning-method choice, Android-version matrix, what-breaks summary, renewal/maintenance. Extends existing `_glossary-android.md#cope` entry with a concrete admin path.

2. **COPE deprecation-rationale doc** — short (400-600 word) doc explaining why COPE is no longer the recommended path, what migration to WPCO looks like, and a "continue running existing COPE fleet" fallback. Replaces Phase 36 COBO §COPE Migration Note forward-link with a full reference doc.

**Gate inputs (research gate must produce):**

- Google AE Help search: `site:support.google.com/work/android "COPE"` — does any page declare deprecation?
- Google Android Developers release notes for Android 16 (2026 target) — does COPE appear in removal list?
- Microsoft Learn Intune Android enrollment docs — does Intune still surface "Corporate-Owned, Work Profile" as an enrollment-profile selector?
- Bayton Android Enterprise FAQ — does the community canonical reference declare COPE deprecated?
- `docs/_glossary-android.md:49` current phrasing — remains "Google recommends WPCO as the successor pattern for the same use case" with "no formal deprecation notice from Google" (HIGH confidence as of 2026-04-21).

**Gate output:** Either (a) `scope = full-admin-guide` OR (b) `scope = deprecation-rationale`. If gate produces ambiguous signal (one source says deprecated, one doesn't), default to (b) deprecation-rationale as the safer scope — a shorter doc that cites the ambiguity is better than a long doc built on contested footing. Mirrors v1.4 CONTEXT D-05 conservative-defer pattern.

**Gate failure recovery:** If gate fails mid-authoring (Google announces deprecation while Phase N plan N-03 is in flight), invoke the Category C-5 escape hatch — re-spawn phase as deprecation-rationale at 40% of original scope, preserving what-was-written as historical appendix. Lock this behavior in phase CONTEXT.

**Do not skip the gate.** Proceeding to plan the full-admin guide without the gate risks publishing "deprecated" language against Google's current positioning (v1.4 banned it for a reason — AEAUDIT-04 literal C1 check) OR publishing a full-admin guide that contradicts a Google deprecation announcement within 6 months.
