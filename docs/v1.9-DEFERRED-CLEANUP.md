## PSSO-FUT-01: NUAL Authorization-Mode MDM Payload Key Literals

**Item omitted:** The MDM plist key literals for **both** NUAL (New User At Login Window) authorization-mode settings -- the one-time "New User Authorization Mode" key (commonly cited as `NewUserAuthorizationMode`) AND the persistent "User Authorization Mode" key -- from the NUAL section of `docs/admin-setup-macos/08-auth-methods-deep-dive.md`. Both display names are documented; neither plist key literal is.

**Omission reason:** LOW research confidence -- the MDM payload key names were not confirmed from an authoritative Settings Catalog payload schema source. The official Intune Platform SSO scenarios documentation shows the Settings Catalog UI display names ("New User Authorization Mode" and "User Authorization Mode", each with supported values Standard / Admin / Groups), but the underlying MDM plist key names for both were not confirmed from the Settings Catalog payload schema or an authoritative Apple MDM specification in this session. The deferral applies symmetrically to both keys.

**What IS documented in guide 08:** The NUAL section documents the behavior, the macOS 14+ version gate, Shared Device Keys enablement, `com.apple.PlatformSSO.AccountShortName` account-name mapping, and the display names of the NUAL Settings Catalog settings (Enable Create User At Login, New User Authorization Mode, User Authorization Mode). The key literal is omitted.

**Where it belongs when resolved:** `docs/admin-setup-macos/08-auth-methods-deep-dive.md`, NUAL section -- specifically in the Settings Catalog settings table and any configuration guidance that currently uses the display name only.

**Requirement linkage:**
- PSSO-11 (deferred arm) -- New User At Login Window documentation; the MDM key literal is the deferred component
- PSSO-FUT-01 -- REQUIREMENTS.md lines 70, 88 -- tracks this as a future verification item

**Resolution trigger:** Verify the exact MDM plist key name (`NewUserAuthorizationMode` or other) against:
1. The live Settings Catalog payload schema in Microsoft Learn or the Intune MDM reference
2. `https://learn.microsoft.com/en-us/intune/device-configuration/settings-catalog/configure-platform-sso-scenarios-macos` (current as of research 2026-06-21)
